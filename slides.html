What does CSS stand for?

Cascading Style Sheets

CSS is a style sheet language that lets you style the elements on your page.

CSS works with HTML, but is not HTML itself.

body {
    background-color: #4a69bd;
    color: #ffffff;
    font-family: 'Barlow', sans-serif;
    margin: 0;
    text-align: center;
}

h1 {
    text-transform: uppercase;
    font-size: 60px;
}

h2 {
    font-size: 40px;
}

The CSS Rule

A block of CSS code is a rule. The rule starts with a selector.
It has sets of properties and values.

Let's use our inspector tool to look at some CSS in the browser!

How can we add CSS to our websites?

Step One

Create a CSS file

Name it something that makes sense to you, with .css at the end

A good choice might be styles.css

Make sure its in the same folder as your HTML file

Step Two: Link Tag

We need to add a link tag in our HTML file

This "links" the CSS file to the HTML file

The link tag belongs in the head of our HTML file

Make sure the href matches what you named your CSS file!

CSS Selectors

So far we have been selecting HTML elements, but what if we want to be more specific?

Example: What if I want to make one of my headers purple but the rest should stay blue?

Remember HTML classes?

<h1 class="main-header">My Favorite Things</h1>

We can use these to apply CSS styles

When using a class selector,
make sure you put a period in front of the class name

.main-header {
    color: blue;
    font-size: 40px;
}

This style will only apply to elements with the class name of "main-header"

What will happen if two elements have the same class name?

Lets practice using class names to apply styles!

We can also use HTML ids for CSS

<h1 id="fav-things">My Favorite Things</h1>

When using an ID selector,
make sure you put a hash in front of the ID name

#fav-things {
    color: blue;
    font-size: 40px;
}

This style will only apply to the single element with the id name of "fav-things"

Remember, an ID can only be one element

Lets practice using ids to apply styles!

The CSS Cascade

What do you think will happen here? Will the H1 elements be blue or green?

h1 {
    color: blue;
}

h1 {
    color: green;
}

ANSWER: green

Since CSS is "cascading", it starts at the top of the stylesheet and moves down.

So, the last declaration will (almost) always win

!important

We can use the !important tag to make sure a style gets applied.

h1 {
    color: blue!important;
}

h1 {
    color: green;
}

In this situation, the cascade would be ignored, and the H1 would be blue

Its not best practice to use important tags; they can make our code hard to understand.

Background Images

We've learned about images with HTML, but we can also use CSS to set a background image

Background Properties

.background-example {
    background-image: url('images/fluffy-cat.jpg');
    background-size: cover;
    height: 300px;
    width: 400px;
}

You can also use a gradient for a background image

Examples here

section {
    background-image: linear-gradient(to top, #30cfd0 0%, #330867 100%);
}

We can specify sides
